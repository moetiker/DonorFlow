1. Update middleware.ts matcher regex to exclude public routes:
   - Change from: `'/((?!api/auth|login|_next/static|_next/image|favicon.ico).*)'`
   - Change to: `'/((?!api/auth|api/public|s/|login|_next/static|_next/image|favicon.ico).*)'`
   - This excludes both `/api/public/*` (API namespace) and `/s/*` (status pages in Phase 3)

2. Add `withPublicApiRoute()` function to lib/api-helpers.ts:
   ```typescript
   /**
    * Wraps public API route handlers with error handling (NO authentication)
    * Use this for routes in /api/public/ namespace only
    */
   export function withPublicApiRoute<T extends any[]>(
     handler: (request: NextRequest, ...args: T) => Promise<NextResponse>
   ) {
     return async (request: NextRequest, ...args: T): Promise<NextResponse> => {
       try {
         return await handler(request, ...args)
       } catch (error) {
         console.error(`Public API Error: ${request.method} ${request.url}`, error)
         return NextResponse.json(
           { error: 'Internal Server Error' },
           { status: 500 }
         )
       }
     }
   }
   ```

Note: Do NOT use `withApiRoute()` - it enforces authentication and would return 401.

  <files>app/api/public/status/[token]/route.ts</files>
  <action>
Create the directory and route file: `app/api/public/status/[token]/route.ts`

Implementation requirements:
1. Use `withPublicApiRoute()` wrapper (NOT withApiRoute)
2. Extract token from params (Next.js 15: params is a Promise, must await)
3. Validate token format: 32 characters (reject others with 404)
4. Get current fiscal year (handle null gracefully)
5. Look up member by statusToken first, then group
6. Use explicit Prisma `select` to prevent PII leakage - ONLY include:
   - Member/Group: id, firstName, lastName, name (for group)
   - Sponsors: id, company, firstName, lastName (NO email, phone, street, postalCode, city, notes)
   - Donations: id, amount, donationDate (NO note)
   - MemberTargets: targetAmount
7. Return 404 for invalid tokens (never 401/403)
8. Calculate progress: target, actual (sum of donations), percentage
9. Format sponsors list with: name, donated (boolean), totalAmount, lastDonation
10. Add security headers: Referrer-Policy: no-referrer, Cache-Control: private, max-age=60
11. Use `serializeDates()` from lib/utils for the response

Response format:
```json
{
  "type": "member" | "group",
  "name": "Display Name",
  "fiscalYear": { "name": "2025/2026", "startDate": "...", "endDate": "..." } | null,
  "progress": { "target": 5000, "actual": 3250, "percentage": 65 },
  "sponsors": [
    { "name": "Company/Person", "donated": true, "totalAmount": 1000, "lastDonation": "..." }
  ]
}
```

For groups: target is 0 (aggregate target calculation deferred to Phase 4).

IMPORTANT: Filter donations by MONETARY type only (not IN_KIND) when calculating amounts.
