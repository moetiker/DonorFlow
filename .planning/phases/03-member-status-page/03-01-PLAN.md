  <action>
Extend the existing public status API to include LYBUNT (Last Year But Unfortunately Not This Year) detection for each sponsor.

Implementation:
1. Query the previous fiscal year (most recent fiscal year BEFORE current fiscal year):
   ```typescript
   const previousFiscalYear = await prisma.fiscalYear.findFirst({
     where: currentFiscalYear ? {
       endDate: { lt: currentFiscalYear.startDate }
     } : undefined,
     orderBy: { endDate: 'desc' },
     select: { id: true }
   })
   ```

2. For member status, include previous year donations in the Prisma query:
   - Add a second donations query with `fiscalYearId: previousFiscalYear?.id`
   - Or query all donations and filter by fiscalYearId in the mapping function

3. Update `calculateSponsorsProgress` function signature to accept `previousFiscalYearId`:
   ```typescript
   function calculateSponsorsProgress(
     sponsors: SponsorWithDonations[],
     previousFiscalYearId: string | null
   )
   ```

4. In the sponsor mapping, add `donatedLastYear` and `isLYBUNT` flags:
   ```typescript
   const donatedLastYear = previousFiscalYearId
     ? sponsor.donations.some(d => d.fiscalYearId === previousFiscalYearId)
     : false
   const donated = sponsor.donations.some(d => d.fiscalYearId === currentFiscalYearId)

   return {
     name: getSponsorDisplayName(sponsor),
     donated,
     donatedLastYear,
     isLYBUNT: donatedLastYear && !donated, // Key flag for UI highlighting
     totalAmount,
     lastDonation
   }
   ```

5. Update the Prisma queries for both member and group to include ALL donations (not just current fiscal year), then filter in the mapping function. This is needed because we need to check both current AND previous year donations.

Keep existing security headers and error handling unchanged.
